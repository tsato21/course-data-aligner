Course Organization System

This comprehensive suite of Google Apps Scripts has been meticulously crafted to automate the standardization and organization of course data within Google Sheets for academic entities. It addresses the common challenge of inconsistent data presentation by individual academic programs, ensuring alignment with centralized database systems. The project comprises a series of specialized scripts, each precisely developed to streamline particular facets of the data management cycle. These include the systematic categorization of course offerings, the synchronization of scheduling data, and the efficient coordination of faculty assignments. The primary objective is to reformulate course-related data to mirror the structured format employed by the institution's database, thereby enhancing administrative efficiency, minimizing the propensity for manual input errors, and ensuring seamless retrieval of well-organized data. You can find the sample sheet from HERE.

Prerequisites

  • A Google account with access to Google Sheets to store and process your course data.
  • An intermediate understanding of Google Sheets functionalities and how to navigate the interface.
  • A foundational understanding of Google Apps Script to understand script functionalities and make any necessary customizations.
  • Basic familiarity with JavaScript programming to interpret the script logic and potentially customize the script to fit specific requirements.

Detailed Descriptions for Each Project

1. Organize Course Offering Data

This script focuses on structuring course offering data from a provided dataset. It filters and restructures the data into the one consistent with the institution's database, ensuring that course codes, semesters, and faculty details are laid out and ready for analysis.

Logic:

  1. The script starts by reading course offering data from the '1_Offering Data by Program' sheet.
  2. It then filters out specific course codes that are predefined and do not require being displayed. These course codes are defined in the constant variables set before the function (e.g., 'MXD304', 'WNV327').
  3. The organized data (Course Code, Semester, Faculty) is then displayed on the '1_Organized Offering Data' sheet.

Usage:

  1. Insert the course offering data, supplied by the target Program, into the orange cells of the '1_Offering Data by Program' sheet.
  2. Navigate to the '1_Organized Offering Data' sheet and click the designated button.
  3. Organized course offering data will be automatically displayed in columns A ~ C.

Note:

  • The function will prompt for the names of the source and display sheets.
  • You can manually change the sheet names as needed.
  • You can also modify the values of the constant variables in `Apps Script` as needed. For example, if there are additional course codes that do not require being displayed, you can add them to the `COURSES_NOT_CHECKED` array.

2-1. Organize Schedule Data (Type 1)

This script is designed to organize class schedule data from a given dataset. It processes and transforms the data to align with the institution's database structure, effectively arranging course codes, days, class numbers, and start and end times for easy analysis.

Logic:

  1. The script starts by reading class schedule data from the '2-1_Schedule Data by Program' sheet.
  2. It checks each cell against a specific pattern (three alphabets followed by three digits). If the cell data matches, the cell data is stored. If it does not match the pattern, it is stored as an exceptional record.
  3. The script then processes grouping data based on course code and day.
  4. Upon grouping, the script examines records with the same day, and merges them, except when the class number includes "2" and "3" due to the lunch break in-between. After this process, the script sorts the organized data by course code.
  5. The organized data for regular courses are then displayed in columns A ~ E in on the '2-1_Organized Schedule Data' sheet, while exceptional cases are listed separately in column G.

Usage:

  1. Insert the class schedule data, supplied by the target Program, into the orange cells of the '2-1_Schedule Data by Program' sheet.
  2. Navigate to the '2-1_Organized Schedule Data' sheet and click the designated button.
  3. Organized class schedule data is automatically displayed in columns A ~ E and G.

Note:

  • The function will prompt for the names of the source and destination sheets.
  • You can manually change the sheet names as needed.
  • The function handles day conversion from English to Japanese.

2-2. Organize Schedule Data (Type 2)

This script is designed to organize class schedule data from a given dataset. It processes and transforms the data to align with the institution's database structure, effectively arranging course codes, titles, faculty names, days, class numbers, and start and end times for easy analysis.

Logic:

  1. The script begins by extracting class schedule data from the '2-2_Schedule Data by Program' sheet.
  2. It then validates each cell's initial element against a specific pattern (three alphabets followed by three digits). If the pattern is not matched, the data is ignored. If it matches, the cell data is parsed into course codes, course titles, and faculties. Any faculty name that matches a pre-defined name in the constant variable is adjusted accordingly.
  3. Next, the script processes the data, generating multiple records if course codes contain "/" or faculties contain ",". It then groups these records based on course code, day, and faculty.
  4. Once grouped, the script scrutinizes records with the same day and merges them, except when the class number includes "2" and "3" due to the lunch break in-between. The data is then organized by course code.
  5. Finally, the processed data is automatically displayed in columns A ~ G on the '2-2_Organized Schedule Data' sheet.

Usage:

  1. Insert the class schedule data, supplied by the target Program, into the orange cells of the '2-2_Schedule Data by Program' sheet.
  2. Navigate to the '2-2_Organized Schedule Data' sheet and click the designated button.
  3. Organized class schedule data is automatically displayed in columns A ~ G.

Note:

  1. The function will prompt for the names of the source and destination sheets.
  2. You can manually change the sheet names as needed.
  3. The function handles day conversion from English to Japanese.
  4. You can also modify the values of the constant variables as needed. For example, if there are additional faculty names that require adjustment, you can add them to the `FACULTY_NAME_REPLACEMENT` array.

2-3. Organize Schedule Data (Type 3)

This Google Apps Script is designed to reformat class schedule data to match the institution's data management systems. It organizes details such as course codes, titles, faculty names, class days and numbers, and start and end times. This provides a simplified and accessible view for further analysis and reporting.

Logic:

  1. Starts by getting schedule data from '2-3_Schedule Data by Program'.
  2. Checks each data entry to make sure it's in the right format, specifically three letters followed by three digits for course codes.
  3. During course record processing, the script checks if the course code includes a slash. If so, it splits the course code and creates two separate records. If the course code is part of a designated category (i.e., EAA course), it retrieves the section code from the second row of the sheet to form the class code. In other cases, it checks if the course code includes a "-", and if not, appends "-1" to form the class code.
  4. When pushing records, the script checks if the target course is in a category that doesn't require a second record. If so, it creates only one record. If the course cell is located in either Monday & Wednesday range or Tuesday & Thursday range, multiple records are created. If it is located in Friday range, only one record is created.
  5. Processed data is displayed in columns A ~ G on the '2-3_Organized Schedule Data' sheet. While displaying data, the script validates each item of the course record. If an item is not valid, it highlights the value in red on the display sheet for easy identification.

Usage:

  1. Start by filling in the '2-3_Schedule Data by Program' sheet with the class schedule data for each program. Make sure that data are input in the range colored in orange.
  2. Next, go to the '2-3_Organized Schedule Data' sheet and click the designated button.
  3. Organized class schedule data is automatically displayed in columns A ~ G.

Note:

  • The script will ask for the names of the source and destination sheets.
  • You can change the sheet names later if you need to.
  • The script can convert day names between English and Japanese.
  • The script is made so you can change parts of it. For example, you can change the `FACULTY_NAME_REPLACEMENT` constant to match the names used at your institution.
